Selection Sort

Similar to bubble sort, but instead of first placing large values into sorted position, it places small values into sorted position.

It works by asking if an item is the smallest it has seen. For each pass, the smallest available value accumulates at the beginning of the array.

Time Complexity is O(n^2) because we still have to compare every number to each other number. Where it is better than bubble sort is the scenario where swaps are for whatever reason costly - selection only swaps at the end of each loop as opposed to bubble which swaps constantly.
