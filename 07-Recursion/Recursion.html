Recursion

Objectives

  • Define what recursion is and how it can be used.
  • Understand the two essential components of a recursive function
  • Visualize the call stack to better debug and understand recursive functions
  • Use helper method recursion and pure recursion to solve more difficult problems.

What is recursion?

Any function that calls itself. Usually done on a smaller and smaller amount of data. Some codebases use it a lot, some not at all. Used under the hood everywhere - JSON.parse / JSON.stringify are usually recursive.

Why use recursion?

Anything that can be solved recursively, can be solved iteratively, usually with a while loop. But some problems lend themselves more to one technique or the other.

The call stack

In almost all programming languages, there is a built in data structure that manages what happens when functions are invoked. The call stack is a stack, and when a function is invoked, it is placed on the top of the stack. When JS sees the return keyword or when a function ends, the compiler will remove the item from the top of the stack.

Recursion takes advantage of this call stack by repeatedly calling the same function over and over with a smaller amount of data. The last item placed onto the stack will be the first to resolve, and then it will pop off, and go to the next one.

Recursive Gameplan

The base case is the condition controlling when the recursion ends. There must be an end, otherwise it would call itself over and over. Often, the base case is an empty list or something similar.
