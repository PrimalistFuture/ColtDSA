Radix Sort

The best possible time for a comparison sort (bubble, quick, merge, selection, insertion) is O(n log n). Except for Radix in edge cases.

RadixSort works on numbers, and works a little differently. Exploits the fact that that information about the size of a number is encoded in the number of digits: the more digits, the bigger the number.

Works by creating buckets for the ending digit of each number. Then the numbers are reordered based on that, and the process repeats for the second to last digit of the number. And so on. Never actually compares the numbers to each other.
