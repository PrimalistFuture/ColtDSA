Analyzing Performance of Arrays and Objects

Objects

    Time of Object actions
  • Insertion - O(1)
  • Removal - O(1)
  • Searching - O(n)
  • Access - O(1)
    Time of Object Methods
  • Object.keys - O(n)
  • Object.values - O(n)
  • Object.entries - O(n)
  • hasOwnProperty - O(1)

When you don't need order, objects are a great type. Items are added using a key. There is no beginning, middle, or end.

Arrays

    Time of Array actions
  • Insertion - depends...
  • Removal - depends...
  • Searching - O(N)
  • Access - O(1)

When inserting items to the end of an array (.push), that is constant. But when adding to the beginning (.unshift)

Same with removing. .pop from the end of the array is faster than .shift from the start of the array.

    Time of Array Methods
  • push - O(1)
  • pop - O(1)
  • shift - O(n)
  • unshift - O(n)
  • concat - O(n)
  • slice - O(n)
  • splice - O(n)
  • sort - O(n * log n)
  • map/filter/reduce - O(n)

When you need order. But this order can come at a time cost when inserting and removing items.
