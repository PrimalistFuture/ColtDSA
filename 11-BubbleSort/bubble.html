General Sort

Sorting is the process of rearranging the items in a collection (like an array) so that the items are in some kind of order. Sorting numbers, alpha, release year, revenue whatever.

Part of the difficulty with sorting algorithms is that there are a bunch of them, and knowing when to use what is a major concern. Though most of them will, when used correctly, end up in the same place.

Sorting is an incredibly common task, so it is good to know how it works. It is important to know what the built-in methods are doing behind the scenes and if you know what sorting technique they employ, you can know what they are good at and what they are bad at.

There is no one size fits all algorithms, though some are generally better than others and some are more niche.

Built-In JS .sort()

The built in JS .sort() method that can be called on arrays doesn't always work the way that you might expect. This is because it turns the array items into strings, then compares them. So things like numbers won't work like you would expect. But letters should.

But we can actually specifiy how we want the method to behave. .sort() accepts an optional comparator function. The comparator looks at pairs of elements (a and b), and determines their sort order based on their return value (a - b).

If (a-b) returns a negative number, a should come before b (ascending order). If (a-b) returns a positive number, a should come after b (descending).

You can define this comparator function normally or as an anonymous callback.

Bubble Sort

Not the most efficient, and not commonly used. But it is a good intro. BubbleSort is an algorithm where the largest values bubble to the top.

Compares the next item to the current item. If the next is smaller than the current, than the items swap. If the items are in the correct order, they do not swap.

After one pass, the largest value will have 'bubbled' to the top. The list then builds from the back.

BubbleSort, and indeed many other sorting algorithms, involve some type of swapping functionality. That logic can be saved to a function and called when needed. CTD.
